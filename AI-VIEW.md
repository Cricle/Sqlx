## Project Overview

### What is Sqlx?

Sqlx is a high-performance, AOT-friendly .NET database access library that uses source generators to generate code at compile time with **zero runtime reflection**.

### Key Statistics
- **3124+ passing tests** (unit + E2E)
- **83.2% branch coverage**
- **Up to 5.8% faster** than Dapper.AOT in some scenarios
- **Zero reflection** - all code generated at compile time
- **Full AOT support** - Native AOT ready

### Supported Databases
- SQLite
- PostgreSQL  
- MySQL
- SQL Server
- Oracle
- DB2

### Core Principles
1. **Zero Reflection** - Everything generated at compile time
2. **Type Safety** - Compile-time validation of SQL and expressions
3. **Performance** - Optimized ResultReader with minimal GC pressure
4. **AOT Compatible** - Full Native AOT support
5. **Developer Experience** - Clean API with IntelliSense support

## Core Architecture

### Component Hierarchy

```
SqlxContext (optional)
    ├── Repository 1 (implements interface)
    │   ├── EntityProvider (column metadata)
    │   ├── ResultReader (DbDataReader → Entity)
    │   └── ParameterBinder (Entity → DbCommand)
    ├── Repository 2
    └── Repository N

All generated by Source Generators at compile time
```

### Code Generation Flow

```
[Sqlx] Attribute
SqlQuery<T> Usage     ──→  Source Generator  ──→  Generated Code
[SqlTemplate] Methods                              ├── EntityProvider
                                                   ├── ResultReader
                                                   ├── ParameterBinder
                                                   └── Repository Implementation
```

## Critical Rules & Gotchas

### ⚠️ MUST FOLLOW

#### 1. SqlDefine Attribute Placement

**✅ CORRECT:**
```csharp
// SqlDefine goes on Repository implementation
[SqlDefine(SqlDefineTypes.SQLite)]
[RepositoryFor(typeof(IUserRepository))]
public partial class UserRepository : IUserRepository { }
```

**❌ WRONG:**
```csharp
// DO NOT put SqlDefine on SqlxContext
[SqlxContext]
[SqlDefine(SqlDefineTypes.SQLite)]  // ❌ WRONG!
public partial class AppDbContext : SqlxContext { }
```

**Why**: SqlxContext is just a container. SQL dialect is configured per-repository.

#### 2. Batch Operations

**✅ CORRECT:**
```csharp
// Use BatchInsertAsync for multiple inserts
await context.Users.BatchInsertAsync(users, cancellationToken);
```

**❌ WRONG:**
```csharp
// Don't loop with InsertAsync
foreach (var user in users)
{
    await context.Users.InsertAsync(user);  // ❌ Inefficient!
}
```

**Why**: BatchInsertAsync generates a single SQL statement for all rows, dramatically improving performance.

#### 3. Output Parameters

**✅ CORRECT:**
```csharp
// Output parameters ONLY in synchronous methods
[SqlTemplate("INSERT INTO {{table}} (name) VALUES (@name)")]
int InsertAndGetId(string name, [OutputParameter(DbType.Int64)] out long id);
```

**❌ WRONG:**
```csharp
// C# doesn't support out/ref in async methods
Task<int> InsertAndGetIdAsync(string name, [OutputParameter] out long id);  // ❌ Won't compile!
```

**Why**: C# language limitation - async methods cannot have out/ref parameters.

#### 4. NULL Handling in Dictionary WHERE

**✅ NEW BEHAVIOR (Current):**
```csharp
var filter = new Dictionary<string, object?> 
{ 
    ["Name"] = "John",
    ["Age"] = null  // Generates: [age] IS NULL
};
// SQL: WHERE ([name] = @name AND [age] IS NULL)
```

**❌ OLD BEHAVIOR (Before recent update):**
```csharp
// NULL values were skipped
var filter = new Dictionary<string, object?> { ["Age"] = null };
// SQL: WHERE 1=1  (null was ignored)
```

**Why**: NULL is a valid search condition. The new behavior generates proper `IS NULL` conditions.

#### 5. SqlTemplate Disambiguation

There are TWO different things called "SqlTemplate":

**1. [SqlTemplate] Attribute** (Sqlx.Annotations)
```csharp
using Sqlx.Annotations;

public interface IUserRepository
{
    // Attribute - marks method for SQL execution
    [SqlTemplate("SELECT {{columns}} FROM {{table}} WHERE id = @id")]
    Task<User?> GetByIdAsync(long id);
}
```

**2. SqlTemplate Class** (Sqlx)
```csharp
using Sqlx;

public interface IUserRepository
{
    // Class - returns template for debugging (doesn't execute)
    [SqlTemplate("SELECT {{columns}} FROM {{table}} WHERE id = @id")]
    SqlTemplate GetByIdSql(long id);
}

// Usage:
var template = repo.GetByIdSql(123);
Console.WriteLine(template.Sql);  // See generated SQL
```

#### 6. Don't Write Raw SQL

**✅ CORRECT:**
```csharp
// Use Sqlx placeholders and methods
[SqlTemplate("SELECT {{columns}} FROM {{table}} WHERE {{where --object filter}}")]
Task<List<User>> SearchAsync(Dictionary<string, object?> filter);
```

**❌ WRONG:**
```csharp
// Don't write raw SQL strings
[SqlTemplate("SELECT id, name, email FROM users WHERE name = @name")]  // ❌ Hardcoded columns
Task<List<User>> SearchAsync(string name);
```

**Why**: Placeholders provide:
- Dialect independence (SQLite vs PostgreSQL vs MySQL)
- Automatic column name mapping
- Type safety
- Maintainability
